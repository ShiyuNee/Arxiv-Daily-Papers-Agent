# Arxiv Daily Papers Agent

每天追踪 arXiv 最新论文，按研究方向自动分类，为每篇论文生成精炼的中文 TL;DR，让你在 10 分钟内掌握当日研究动态。

> **每日更新**：我们每天自动整理 cs.CL 最新论文，无需任何配置或 LLM 调用，直接查看本仓库对应日期目录下的报告即可。如果你没有 LLM token，或者只是想快速浏览，直接看 `cs.CL_YYYY-MM-DD/` 目录下的 Markdown 报告就行。

## 为什么做这个工具

arXiv 每天更新大量论文，cs.CL 一个类别一天就有 100-200 篇。逐篇阅读摘要既耗时又难以把握整体趋势。这个工具的目标是：

- **开箱即用**：每天自动生成整理好的报告，直接阅读，无需自己跑流程
- **快速概览**：不是逐篇翻译摘要，而是用两句话（问题 + 方法）精炼概括每篇论文的核心贡献
- **按方向归类**：将论文按研究主题分组，一眼看出今天哪些方向最活跃
- **发现趋势**：每个 topic 附带当日研究动态总结，提炼共性、发现新趋势

## 核心功能

### 1. 论文数据采集

通过 [papers.cool](https://papers.cool) 爬取指定 arXiv 类别和日期的论文信息：

- 标题、作者、摘要、分类、发布时间、arXiv ID、PDF 链接
- 输出结构化 JSON + 原始 Markdown，支持离线复用
- 支持所有 arXiv 类别（cs.CL、cs.IR、cs.LG、cs.CV 等）

### 2. 智能分类

基于论文摘要将论文归入预定义的研究主题（Topic）：

- **30+ 预定义 Topic**：覆盖 Agent/RL、大模型核心、NLP 基础任务、检索、多模态、数据与知识等方向（详见 `TOPICS.md`）
- **动态扩展**：当论文无法归入现有 Topic 时，自动创建新 Topic 并记录
- **两级分类**：先关键词粗筛（`classify_papers.py`），再人工精调，确保分类质量

### 3. 高质量中文 TL;DR

为每篇论文生成两句话的中文精炼摘要，遵循严格的质量标准：

```
TL;DR: 问题：[具体痛点 + 为什么现有方法不行] 方法：[方法名 + 核心技术机制 + 关键结果]
```

**质量要求**：
- 问题部分：说清什么场景下、什么现有方法有什么具体缺陷、为什么不行
- 方法部分：说清方法叫什么、具体怎么做（不只是方法类型）、关键结果或约束

**示例对比**：

| 质量等级 | TL;DR |
|---------|-------|
| 高质量 | 问题：视觉语言模型存在"视觉不接地"问题——可能完全靠语言先验而非图像回答，现有置信度方法无法检测。方法：提出BICR，在训练时对比真实图像和遮蔽图像的隐藏状态，用排序损失惩罚对遮蔽视图的高置信度，零额外推理成本下实现跨模型最佳校准。 |
| 低质量 | 问题：研究精度问题。方法：提出新的训练策略。 |

### 4. 研究动态总结

为每个 Topic 归纳当日研究动态（2-4 句话），包括：

- 该方向关注什么问题
- 有什么新进展（方法创新、新发现、新视角）
- 趋势观察

### 5. 输出报告

生成结构化的 Markdown 报告，包含：

- **总览表格**：一目了然各 Topic 的论文数和核心关键词
- **分类详情**：每个 Topic 的研究动态 + 论文列表（含 TL;DR）
- **排除标注**：多语言/医学类论文标注排除原因但仍保留在报告中
- **统计信息**：论文总数、各 Topic 分布、新增 Topic 记录

## 自己生成报告

如果你想整理其他类别或日期的论文（如 cs.CV、cs.LG），可以用 AI Agent 自行生成。整个流程由 Agent 自动完成，无需手动分步操作。

### 环境依赖

```bash
pip install requests beautifulsoup4
```

### 使用 AI Agent 生成

在支持工具调用的 AI Agent（如 Cursor、Claude Desktop 等）中，将 `SKILL.md` 配置为 Agent 指令，然后发送：

```
整理 https://papers.cool/arxiv/cs.CL?date=2026-05-13&show=300
```

Agent 会自动完成：爬取数据 → 读取分类体系 → 通读分类 → 撰写 TL;DR → 生成报告。

### 支持的 arXiv 类别

| 类别 | 说明 | 类别 | 说明 |
|------|------|------|------|
| cs.CL | 计算语言学 | cs.CV | 计算机视觉 |
| cs.IR | 信息检索 | cs.AI | 人工智能 |
| cs.LG | 机器学习 | cs.SE | 软件工程 |
| cs.HC | 人机交互 | cs.RO | 机器人学 |

## 工作流概览

```
papers.cool URL
       │
       ▼
┌──────────────────────┐
│  papers_cool_crawler │  爬取论文数据
│  (Python 脚本)        │  → JSON + Markdown
└──────────┬───────────┘
           │
           ▼
┌──────────────────────┐
│  classify_papers.py  │  关键词预分类（可选）
│  (Python 脚本)        │  → _classified.json
└──────────┬───────────┘
           │
           ▼
┌──────────────────────┐
│  AI Agent + SKILL.md │  通读论文 → Topic 分类
│  (LLM 驱动)           │  → 研究动态总结
│                      │  → 中文 TL;DR
│                      │  → 结构化报告
└──────────┬───────────┘
           │
           ▼
  {category}_{date}/summary.md  最终输出报告
```

## 持续优化

本仓库的流程和规范会根据每日整理的反馈持续迭代：

- **分类体系演进**：遇到无法归入现有 Topic 的论文时，自动扩充 `TOPICS.md`，分类粒度随论文趋势不断精细化
- **TL;DR 质量提升**：基于每日输出的质量检查，持续优化 `SKILL.md` 中的撰写规范和示例，减少空泛表述
- **流程改进**：爬虫、分类脚本、Agent 指令均根据实际运行反馈迭代，确保稳定性和输出质量
